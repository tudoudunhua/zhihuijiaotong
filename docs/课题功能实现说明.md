# 智慧交通课题功能实现说明

本文档说明系统针对课题要求的改进与已实现功能，便于论文撰写与答辩说明。

---

## 一、多源数据「接不通、用不了」

- **统一接口**：摄像头、违章等已通过 SpringBoot REST 接口统一接入（`/api/cameras`、`/api/violations` 等）。
- **数据清洗**：违章新增/编辑时进行合理性校验：
  - 罚款金额：0～10000 元；
  - 扣分：0～12 分；
  - 车牌号长度校验。
- **分层存储**：历史违章存 MySQL；若后续接入车流量等实时数据，可扩展 Redis 缓存（如最近 5 分钟数据）。

---

## 二、违章识别与预警「看不准、漏得多」

- 现有 **智能预警** 模块基于历史数据与 ML 预测（`MIPredictService`），用于趋势预测与风险等级。
- **四级预警** 已实现（见下）。

---

## 三、预警信息「推不对、传不快」

### 1. 分级推送（四级预警）

- **蓝**：小违章/低风险 → 推送对象：路段网格员  
- **黄**：中低 → 路段网格员、片区负责人  
- **橙**：中高 → 交警中队  
- **红**：大事故/高风险 → 交警支队、消防  

规则在 `WarningService.applyLevelAndPushTarget()` 中，按 `severity` 与 `count` 自动定级并写入 `level`、`pushTarget`。

### 2. 实时推送

- **WebSocket**：后端使用 Spring WebSocket（STOMP + SockJS），端点 `/ws`，主题 `/topic/warnings`。
- 新预警写入数据库后立即通过 WebSocket 推送给前端。
- 前端「预警列表」页连接 WebSocket，收到新预警时弹窗提示并刷新列表。

## 四、数据安全与隐私

### 1. 车牌脱敏

- **规则**：保留首 2 位与最后 1 位，中间用 `*` 代替，例如：京A12345 → 京A****5。
- **实现**：`PrivacyUtil.maskPlateNumber()`；接口返回使用 `ViolationDTO`，按当前用户角色决定是否脱敏。
- **权限**：仅 **管理员（ADMIN）** 可见完整车牌；非管理员（交管人员、普通查看者）仅见脱敏车牌。角色从 JWT 的 `role` 解析。

### 2. 权限控制

- **JWT 含 role**：登录时生成 token 写入 `role`（`AuthController`、`JwtUtil`）。
- **前端**：根据 `user.role` 控制菜单，「添加记录」「摄像头管理」仅管理员可见；其余角色可见首页、违章记录、统计分析、智能预警、预警列表等。
- **后端**：违章列表/详情根据 JWT 中的 role 决定返回脱敏或完整车牌。

### 3. 传输与存储

- 生产环境建议：**HTTPS** 传输；敏感字段可扩展 **AES 加密** 落库（当前为脱敏展示，未实现库内加密，可作后续扩展）。

## 六、使用与验证建议

1. **角色**：在 `sys_user` 中为测试用户设置 `role` 为 `ADMIN` 或 `USER`，验证管理员可见完整车牌、非管理员见脱敏。
2. **预警**：触发一次高峰预警或 ML 预警（如调用 `/api/intelligent/trigger-ml` 或等待定时任务），在「预警列表」页查看蓝/黄/橙/红及推送对象；若前端已连 WebSocket，将收到实时弹窗。
3. **数据清洗**：新增违章时输入罚款 15000 或扣分 15，应被后端拒绝并返回错误提示。
